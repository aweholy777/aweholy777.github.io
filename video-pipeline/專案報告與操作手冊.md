> 建立：2026-06-10 ~ 2026-06-11｜狀態：全自動運行中
> 用途：日常查詢、故障排除、未來套用到其他專案、決策追溯

## 一、專案總覽

**目標**：把 cmtc.tw 的 3,477 篇每日 QT 靈修文章（新約 1,495＋舊約 1,982）做成「有人朗讀」的影片，上傳 YouTube 並嵌回文章頁。

**成果：三層產品線**

| 產品線 | 形式 | 產能 | 執行機器 |
|--------|------|------|----------|
| 靜態朗讀版 | 背景圖＋台語女聲＋同步字幕，1080p | ~35 秒/篇，全集一天內 | MS-S1 MAX（CPU） |
| 數位主播版 | AI 主播全程開口講述（嘴型同步） | ~14 小時/篇，一晚一篇 | Windows 3060（GPU） |
| YouTube 發布 | 自動上傳＋嵌入文章＋網站部署 | 每天中午自動 | Windows → GitHub |

**旁白規則**：開頭念「YYYY年M月D日，每日QT。今天的經文進度是：書卷X章Y到Z節」→ 跳過經文引用 → 直接從「今天默想經文」開始念到「今天的回應」結束。

**生產順序**（2026-06-11 定案）：新約書卷順（馬太福音 1 章起）→ 新約完才換舊約；同段經文多年份只做日期最新那篇；去重後共 2,678 篇。

## 二、系統架構

```
┌─ Windows 3060（主機）────────────────────────────┐
│ 21:00 排程 nightly_head.py                        │
│   → 按書卷隊列挑一篇 → edge-tts 產音訊+字幕      │
│   → ComfyUI(本機8188) InfiniteTalk 生成主播影片  │
│   → ffmpeg 放大1080p+燒字幕 → video-output/head/ │
│ 12:30 排程 yt_publish.py --auto                   │
│   → 上傳 YouTube → 嵌入主資料夾 md（不碰git）    │
│   → 在發布副本嵌入+commit+push → 網站自動部署    │
└──────────────────────────────────────────────────┘
┌─ MS-S1 MAX（Ubuntu, 192.168.68.61）──────────────┐
│ ~/qtwork 靜態版批次（CPU, edge-tts+ffmpeg）       │
│ ComfyUI(8188) 備用（ROCm 跑擴散慢，不當主播工人）│
│ crontab @reboot 自動恢復                          │
└──────────────────────────────────────────────────┘
┌─ 發布副本 C:\Users\aweholy\qt-publish\ ──────────┐
│ 乾淨 clone，只跟 GitHub 同步                      │
│ 自動 push 只在這裡做，與主資料夾的               │
│ 3,836 筆未整理變更完全隔離                        │
└──────────────────────────────────────────────────┘
```

## 三、關鍵決策記錄（追溯用）

| 決策 | 原因 |
|------|------|
| TTS 用 Edge TTS（zh-TW-HsiaoChenNeural 女聲, +10%） | 免費、無限量、台灣腔品質佳；3,477 篇用付費 TTS 成本數百美金起 |
| 字幕用「音訊長度按字數比例」估算 | edge-tts 7.x 不回傳逐字時間戳；估算誤差 ±1 秒可接受 |
| 數位主播用 InfiniteTalk（Wan2.1 I2V 480P fp8） | 唯一支援「單圖＋音訊→不限長度」的開源方案；ComfyUI 有官方整合 |
| 3060 用 640×368＋blockswap 40 | 832×480 會撐爆 12GB VRAM 觸發系統記憶體回退，慢 16 倍；這是 12GB 卡的甜蜜點 |
| 必裝 lightx2v 加速 LoRA | 讓 6 步采樣成立；沒有它要 25+ 步，慢 4 倍 |
| MS-S1 MAX 不跑數位主播 | 實測 ROCm 跑擴散 5.4 分/步，比 3060 慢 3~4 倍；它的強項是 CPU/大記憶體 |
| 工作流用 UI 格式＋執行時自動轉 API | comfy_talking_head.py 內建轉換器（讀 /object_info），改工作流不用手動匯出 |
| 自動 push 用獨立發布副本 | 主資料夾有 3,836 筆未整理變更＋otqt 資料夾被程式鎖住，直接 push 有資料風險 |
| YouTube 一天一支不申請配額 | API 免費配額 1,600 點/支、10,000 點/天 ≈ 6 支；一晚一篇遠低於上限 |

**實測數據**：3060 生成速率 ≈ 1.6~2 分鐘運算/秒影片（45~58 秒/步）；6.7 分鐘文章 ≈ 10.8 小時。靜態版 ≈ 35 秒/篇。全程主播做完 2,678 篇 ≈ 7 年（不可行），故主播版按書卷順序慢慢累積，靜態版先覆蓋全集。

## 六、故障排除（本專案實際踩過的坑）

1. **生成極慢（>5分/步）＋GPU 100% 但功耗低**：VRAM 爆了觸發系統記憶體回退。降解析度或加大 blockswap。3060 12GB 必須 640×368。
2. **批次檔中文變亂碼、指令解析失敗**：cmd 用 CP950 讀批次檔。`.bat` 一律寫純英文，中文輸出用 PowerShell＋`-Encoding utf8`。
3. **Python subprocess 讀 ffmpeg 輸出炸 UnicodeDecodeError**：subprocess.run 要加 `encoding="utf-8", errors="replace"`。
4. **ssh 遠端 pkill 把自己殺了**：pkill 的 pattern 會匹配到自己的命令字串。用 `pkill -f '[m]ain.py'` 寫法，或把邏輯放進遠端腳本檔再執行。
5. **PowerShell 5.1 不吃新語法**：不能用三元運算子 `?:`；ps1 內含中文要小心編碼（UTF-8 無 BOM 會亂）→ ps1 盡量純 ASCII。
6. **ComfyUI 例行節點未裝卻回 200**：`/object_info/<節點>` 對不存在的節點回空 `{}` 不是 404，要檢查內容非空。
7. **git 動不了：index.lock / Permission denied**：先刪 .git\index.lock；otqt 資料夾曾被程式鎖住（疑防毒/索引），這是改用發布副本的原因之一。
8. **edge-tts 沒有逐字時間戳**：7.x 不一定回傳 WordBoundary，pipeline 已內建估算 fallback，無需處理。
9. **YouTube OAuth 七天過期**：OAuth 應用要「發布」成正式版（Google Auth Platform → 目標對象 → 發布應用程式），否則 refresh token 一週失效。
10. **ComfyUI 工作流模型檔名**：模型放錯資料夾或檔名帶子目錄前綴（`wanvideo\`）會驗證失敗，依錯誤訊息列出的合法清單修正。

## 七、安全須知

- `client_secret.json`、`yt_token.json` 已加入 .gitignore，**絕不可進公開 repo**。外洩要立即到 Google Cloud Console 撤銷重發。
- SSH 私鑰在 `%USERPROFILE%\.ssh\qt_lan_key`，只授權到 MS-S1 MAX。
- AI 助手不代輸入任何密碼/不代登入帳號——Google 授權、SSH 首次密碼都由本人操作（本專案即如此執行）。
- 發布副本只做「嵌入＋push」，絕不放任何憑證。

## 八、重建指南（換機器/災難復原）

1. **Windows 生成機**：裝 Python3、ffmpeg、Comfy Desktop → 跑 `setup_infinitetalk.bat`（裝節點＋下載 31GB 模型）＋ `download_clipvision.bat` ＋ `download_lora.bat` → 主播圖放 `video-pipeline\assets\presenter.png` → `run_smoke.bat` 驗證 → `setup_schedules.bat` 建排程。
2. **Ubuntu 靜態機**：開 SSH → 跑 `setup_lan_infinitetalk.sh`（若也要 GPU 能力）→ `deploy_lan_static.bat` 部署靜態批次 → `setup_autorecover.bat` 設開機恢復。
3. **YouTube**：Google Cloud 建專案 → 啟用 YouTube Data API v3 → OAuth（外部＋發布）→ 桌面版用戶端 JSON 存 `video-pipeline\client_secret.json` → 首跑授權一次。
4. **發布副本**：`setup_publish_clone.bat`。
5. 模型清單與來源寫在 `setup_infinitetalk.bat` 內（HuggingFace Kijai/WanVideo_comfy ＋ Comfy-Org repackaged）。

## 十、可套用到其他專案的模式

1. **軍師/士兵派工**：規劃用聰明模型，出工用便宜/免費資源；result.md 落地、git diff 驗收、上下文不灌爆（見 CLAUDE.md）。
2. **批次檔遙控模式**：AI 透過「寫 bat/ps1 → 執行 → 讀輸出檔」操作 Windows；輸出一律落地成檔案再讀，繞過終端機限制。
3. **SSH 金鑰遙控模式**：金鑰由 AI 生成、密碼授權由人做一次，之後 AI 全自動遠端管理 Linux 主機。
4. **冪等批次設計**：所有批次「已存在即跳過」，當機/中斷直接重跑同指令。
5. **發布副本隔離**：工作目錄髒亂時，自動發布走獨立乾淨 clone，風險歸零。
6. **決策關卡**：大規模量產前先做單件實測計時，用真實數據定範圍（本專案靠這個避開了「7 年工期」的坑）。
