# 本 repo 派工分析報告

掃描日期：2026-06-10。依影片原則（規劃用聰明 AI、出工用免費大軍）分類目前的工作。

## 現況

repo 有約 3,832 筆未提交變更：約 3,512 筆刪除（`content/daily-qt/2016-2017/` 等舊目錄重整搬遷）、310 筆修改、10 筆未追蹤。內容主體是 ntqt 約 1,496 篇＋otqt 約 1,967 篇 QT 文章，外加按書卷排序的 `_index.md` 索引、R2 圖片遷移腳本、Hugo 設定與 GitHub Actions 部署。

## 第零步（派工前必做，軍師親自）

先把目前 3,800+ 筆變更整理提交，建立乾淨基線。沒有乾淨基線，`git diff` 驗收就失效——士兵的改動會跟舊變更混在一起分不出來。這是目前最優先的軍師工作。

## 適合派給士兵的工作（DeepSeek V4 Flash）

這類工作量大、有固定模板、規格寫清楚後任何模型都做得好，正是「輸出燒大量 token」該外包的部分：

1. **QT 文章格式正規化**（約 3,463 篇）：統一 front matter、日期格式（檔名 `YYYY-MM-DD.md` vs 標題全形破折號）、四段結構檢查。拆成 ntqt/otqt 各按年份分包，可 7 路平行。
2. **書卷索引重建**：`_index.md` 目前手工維護、有重複條目風險（同經文多年份）。派工逐書卷重建，每個書卷一個子任務，平行跑。`sort-daily-qt-indexes.js` 的邏輯可寫進 plan.md 當規格。
3. **圖片 URL 批次遷移收尾**：`update-image-urls.py` 已有，但 310 筆修改檔案中可能還有殘留的本地 `/images/` 路徑，派士兵全量掃描替換並回報清單。
4. **死連結與孤兒頁掃描**：3,500 篇互相連結的文章重整後必有斷鏈，派士兵掃描並產出 result.md 清單，軍師只看清單決定怎麼修。
5. **`_index - 複製.md` 之類的垃圾檔清理**：掃描全站重複/備份檔，列清單給軍師確認後批次刪除。
